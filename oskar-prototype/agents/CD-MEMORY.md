## IMAGE STATUS TAGS

When evaluating images in IMAGES.md, ALWAYS include a `**Status:**` field. This is how the system knows what to do with the image.

| Tag | Meaning | Used For |
|-----|---------|----------|
| `INGESTED` | Uploaded image has been evaluated | Original uploads after CD analysis |
| `✓ HERO` | THE primary image for a vibe's hero section | 4 vibes = max 4 HERO images. One per vibe. |
| `✓ APPROVED` | Good image ready to use for a section | Menu, location, evening programs, residents, etc. |
| `B-ROLL` | Usable but not primary | Alternate versions, decent but not best |
| `✗ TRASH` | Failed generation, do not use | Wrong subject, wrong color, wrong scale |
| `PENDING` | Not yet evaluated | Newly generated, awaiting CD review |

**CRITICAL:** When evaluating a generated image, you MUST add:
1. `**Status:**` field with appropriate tag
2. `**CD Analysis:**` field with your evaluation

Without these fields, the image appears in Uploads instead of Image Library.

**Example entry for evaluated generated image:**
```
### shared-location-section-...-4.jpg
**Generated:** 21:31:20
**Status:** ✓ HERO
**CD Analysis:** THE DEFINITIVE SHOT. Blue hour lighting, Falcon's Flight climbing the cliff, café visible on top. 200m scale is felt. This is the Location section hero for all vibes.
```

## CREATIVE BRIEF — COLOR FORMAT (MANDATORY)

When writing colors in CREATIVE-BRIEF.md, you MUST use this exact format with FOUR colors:

```markdown
**Colors:**
- Primary: `#8B4513` (Saddle Brown — dallah brass, warm wood)
- Secondary: `#F5F5DC` (Beige — sand, cream, cushions)
- Accent: `#722F37` (Burgundy — kilim accents, shemagh details)
- Text: `#2C1810` (Deep Brown — warm readable text)
```

**THE FOUR COLOR ROLES:**
| Role | Purpose | Example |
|------|---------|---------|
| **Primary** | Main brand color, backgrounds, large areas | `#8B4513` — Saddle Brown |
| **Secondary** | Supporting color, secondary backgrounds | `#F5F5DC` — Beige |
| **Accent** | Highlights, CTAs, emphasis | `#722F37` — Burgundy |
| **Text** | Body text, readable contrast | `#2C1810` — Deep Brown |

**CRITICAL RULES:**
1. ALWAYS include all 4 colors (Primary, Secondary, Accent, Text)
2. ALWAYS use backticks around hex codes: `` `#RRGGBB` ``
3. ALWAYS include a description in parentheses
4. The parser extracts colors using this exact format
5. If you forget the Text color, the UI will show default/fallback colors

**WHY THIS MATTERS:**
The vibe card gallery displays these 4 colors as swatches. If you only write 3 colors, the parser can't find the 4th, and the UI falls back to hallucinated defaults. This is a Darth Hallucinator bug waiting to happen.

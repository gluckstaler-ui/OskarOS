## IMAGE STATUS TAGS

When evaluating images in IMAGES.md, ALWAYS include a `**Status:**` field. This is how the system knows what to do with the image.

| Tag | Meaning | Used For |
|-----|---------|----------|
| `INGESTED` | Uploaded image has been evaluated | Original uploads after CD analysis |
| `✓ HERO` | THE primary image for a vibe's hero section | 4 vibes = max 4 HERO images. One per vibe. |
| `✓ APPROVED` | Good image ready to use for a section | Menu, location, evening programs, residents, etc. |
| `B-ROLL` | Usable but not primary | Alternate versions, decent but not best |
| `✗ TRASH` | Failed generation, do not use | Wrong subject, wrong color, wrong scale |
| `PENDING` | Not yet evaluated | Newly generated, awaiting CD review |

**CRITICAL:** When evaluating a generated image, you MUST add:
1. `**Status:**` field with appropriate tag
2. `**CD Analysis:**` field with your evaluation

Without these fields, the image appears in Uploads instead of Image Library.

**Example entry for evaluated generated image:**
```
### shared-location-section-...-4.jpg
**Generated:** 21:31:20
**Status:** ✓ HERO
**CD Analysis:** THE DEFINITIVE SHOT. Blue hour lighting, Falcon's Flight climbing the cliff, café visible on top. 200m scale is felt. This is the Location section hero for all vibes.
```

## CREATIVE BRIEF — COLOR FORMAT (MANDATORY)

When writing colors in CREATIVE-BRIEF.md, you MUST use this exact format with FOUR colors:

```markdown
**Colors:**
- Primary: `#8B4513` (Saddle Brown — dallah brass, warm wood)
- Secondary: `#F5F5DC` (Beige — sand, cream, cushions)
- Accent: `#722F37` (Burgundy — kilim accents, shemagh details)
- Text: `#2C1810` (Deep Brown — warm readable text)
```

**THE FOUR COLOR ROLES:**
| Role | Purpose | Example |
|------|---------|---------|
| **Primary** | Main brand color, backgrounds, large areas | `#8B4513` — Saddle Brown |
| **Secondary** | Supporting color, secondary backgrounds | `#F5F5DC` — Beige |
| **Accent** | Highlights, CTAs, emphasis | `#722F37` — Burgundy |
| **Text** | Body text, readable contrast | `#2C1810` — Deep Brown |

**CRITICAL RULES:**
1. ALWAYS include all 4 colors (Primary, Secondary, Accent, Text)
2. ALWAYS use backticks around hex codes: `` `#RRGGBB` ``
3. ALWAYS include a description in parentheses
4. The parser extracts colors using this exact format
5. If you forget the Text color, the UI will show default/fallback colors

**WHY THIS MATTERS:**
The vibe card gallery displays these 4 colors as swatches. If you only write 3 colors, the parser can't find the 4th, and the UI falls back to hallucinated defaults. This is a Darth Hallucinator bug waiting to happen.

**THE TWO FONT ROLES:**
| Role | Purpose | Example |
|------|---------|---------|
| **Headings** | Headlines, titles, display text | Playfair Display |
| **Body** | Paragraphs, descriptions, readable text | Source Sans Pro |

**EACH VIBE MUST HAVE DIFFERENT FONTS:**

| Vibe | Headings Font | Body Font | Why |
|------|---------------|-----------|-----|
| QAHWA | Playfair Display | Source Sans Pro | Elegant heritage, warm |

| THE RACE | Oswald | Roboto | Bold, athletic, compressed |
| MAJLIS | Cormorant Garamond | Libre Baskerville | Refined luxury, traditional |

**CRITICAL RULES:**
1. ALWAYS include both Headings and Body fonts
2. Each vibe MUST have DIFFERENT fonts — don't use the same fonts for all vibes

**WHY THIS MATTERS:**
The vibe cards display the vibe name using the Headings font and the "who it's for" using the Body font. If all vibes use the same fonts, they look identical and the visual differentiation is lost.

## VIBE PREVIEW SECTION (FOR GALLERY CARDS)

When multiple HTML versions exist for the same base vibe, you MUST add a `## Vibe Preview` section to CREATIVE-BRIEF.md. This section maps **each HTML file** to its gallery card metadata.

**Format:**

```markdown
## Vibe Preview

Gallery cards for vibe selection. Each entry maps to a built HTML file.

### vibe-1-qahwa.html
**Name:** QAHWA — Original
**Audience:** Heritage Seekers & Homesick Families
**Mood:** Warm, Nostalgic, Guilt-Inducing
**Colors:**
- Primary: `#8B4513`
- Secondary: `#F5F5DC`
- Accent: `#722F37`
- Text: `#2C1810`
**Fonts:**
- Headings: Playfair Display
- Body: Source Sans Pro

### vibe-1-qahwa-v2.html
**Name:** QAHWA — Refined
**Audience:** Heritage Seekers & Homesick Families
**Mood:** Warm, Nostalgic, Guilt-Inducing
**Colors:**
- Primary: `#8B4513`
- Secondary: `#F5F5DC`
- Accent: `#722F37`
- Text: `#2C1810`
**Fonts:**
- Headings: Playfair Display
- Body: Source Sans Pro
```

**CRITICAL RULES:**

1. **Section header is a filename** — Use `### filename.html` exactly matching the HTML file
2. **Name field differentiates versions** — "QAHWA — Original" vs "QAHWA — Refined" vs "QAHWA — Grandmother's House"
3. **Audience is brand-speak** — Short persona (5-7 words): "Heritage Seekers & Homesick Families"
4. **Mood is adjectives only** — 3-5 adjectives, comma-separated: "Warm, Nostalgic, Guilt-Inducing"
5. **One entry per HTML file** — Even if colors/fonts are the same, list every file

**WHY THIS MATTERS:**

The gallery shows ALL built HTML files as selectable vibes. Without this section:
- Gallery cards show generic fallback data
- User can't tell versions apart
- Selection becomes confusing

**WHEN TO USE:**

Add Vibe Preview section when:
- You have multiple HTML versions of a vibe (v1, v2, refined, etc.)
- The `# VIBE N:` sections describe the concept, but multiple HTML files exist
- WebDev has built variations you want to distinguish

**ARCHITECTURAL LESSONS:**

1. **Brunnen = Template.** Every project page inherits from Brunnen, but adapts:
   - Sursee: Mini-gallery + Fassadenstudien (it's a quartier)
   - Bottmingen: Compact — 3 Grundrisse, no mini-gallery (small MFH)
   - Basel Loggia: Diagram-pair (it's a study, not a built project)

2. **No image duplication.** Each image has ONE purpose, ONE location on ONE page. If you need to use it twice, you're making a mistake.

3. **Section types are intentional:**
   - `grundrisse-section` = floor plans, technical drawings (`object-fit: contain`, white bg)
   - `mini-gallery` = photographs, renderings (`object-fit: cover`, 4/3 aspect)
   - `diagram-pair` = conceptual comparisons (2-column, explainer below)
   - `story-pair` = narrative sections (image + text, reversible)

4. **The user's hierarchy of content:**
   - Hero = the ONE defining image
   - Story-pairs = narrative flow, each with unique image
   - Grundrisse = technical documentation
   - Each section serves a different communication purpose

5. **Hero overlay values matter:**
   - Brunnen: `rgba(0,0,0,0.45)` — competition winner gets drama
   - Sursee/Bottmingen: `rgba(0,0,0,0.55)` — quartier/compact needs more contrast
   - Basel Loggia: `rgba(0,0,0,0.45)` — diagram hero is lighter

6. **CSS variables are shared.** All four pages use the same color palette. Changes to one affect the system.
